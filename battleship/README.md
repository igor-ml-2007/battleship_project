# Battleship

---

## ИДЕЯ

Классическая консольная игра **"Морской бой"**: игрок расставляет корабли, играет против ИИ, сохраняет партию и ведет статистику побед/поражений.

Основной сдаваемый вариант - кроссплатформенный консольный `C++20`-проект. Дополнительно в репозитории есть оконный macOS-интерфейс, но для проверки на любой ОС используется `battleship_console`.

## ФИЧИ

### 1) Классический игровой процесс

* Поле 10x10
* Стандартный набор кораблей: 4, 3, 3, 2, 2, 2, 1, 1, 1, 1
* Пошаговая игра игрок vs ИИ
* Попадание оставляет ход за стреляющим
* Победа определяется уничтожением всего флота

---

### 2) Расстановка кораблей

* Ручная расстановка командой вида `A1 h` или `A1 v`
* Автоматическая расстановка командой `auto`
* Проверка выхода за границы поля
* Проверка пересечений кораблей
* Проверка правила, что корабли не должны стоять рядом

---

### 3) Консольный интерфейс

* Стартовое меню: новая игра, продолжить, выйти
* Отображение своего поля
* Отображение радара противника
* Журнал последних событий партии
* Понятные команды для подготовки и боя

---

### 4) Умный ИИ

* Минимум 2 стратегии ИИ
* `random` - случайные выстрелы по неизвестным клеткам
* `hunt` - поиск по шахматному паттерну и добивание найденного корабля
* Strategy-паттерн позволяет добавить новую стратегию без переписывания `Match`

---

### 5) Сохранение / загрузка

* Сохранение партии в `battleship_save.txt`
* Загрузка через пункт `Продолжить`
* Сохраняются поля, фаза игры, очередь оставшихся кораблей, ход, журнал и состояние ИИ
* Статистика побед/поражений хранится в `battleship_stats.txt`

---

### 6) Архитектура и ООП

* Разделение на `domain`, `application`, `infrastructure`, `console`
* Бизнес-логика отделена от консольного UI
* Dependency Injection через интерфейсы
* SOLID и GRASP: High Cohesion, Low Coupling
* GoF-паттерны: Strategy, Factory, Repository, Facade

---

### 7) Современный C++

* RAII и rule of zero
* `std::unique_ptr`, `std::shared_ptr`, отсутствие ручного `new/delete`
* Function templates: `countWhere`, `joinMapped`
* Class templates: `BoundedHistory<T>`, `Generator<T>`
* Lambdas со стандартными алгоритмами
* Move-семантика для стратегий, сервисов и объектов состояния
* `auto`, `std::optional`, `std::deque`, `std::vector`
* Exceptions там, где ошибка действительно аварийная

---

### 8) Асинхронность и корутины

* `std::async` используется для асинхронного сохранения статистики
* Синхронизация выполняется через `std::future::get`
* Coroutine-генератор `standardFleetPlan()` через `co_yield` формирует стандартный набор кораблей

---

### 9) Тестирование

* Функциональные тесты в `battleship_tests`
* Проверка авторасстановки флота
* Проверка выстрела за пределы поля
* Проверка сохранения/загрузки состояния
* Проверка случайной стратегии ИИ
* Проверка template-хелпера `countWhere`

## ПРИМЕР ИСПОЛЬЗОВАНИЯ

```text
1. Новая игра
2. Выбрать ИИ: 2
3. auto
4. start
5. A1
6. B4
7. save
8. quit
```

Ручная расстановка:

```text
A1 h
C1 h
E1 v
clear
auto
start
```

Обозначения:

* `.` - пустая или неизвестная клетка
* `#` - ваш корабль
* `o` - промах
* `X` - попадание
* `Z` - потопленная часть корабля
